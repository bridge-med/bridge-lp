# CxO Roadmap Log

日々の現場仕事・数字成果・組織改善を、**将来の CxO 候補（COO / 事業責任者 / 医療機関運営責任者）としての実績ログ**に変換する個人用アプリ。

医療機関の経営支援・クリニック運営支援・リハ部門改善・採用・PMI・施設基準/加算・業務改善といった本業の現場仕事を、

> どの現場課題に → 何の数字に → どんな打ち手で → Before/After → 経営的な意味 → 再利用できる型 → キャリア表現

の粒度で記録し、職務経歴書 / LinkedIn / 昇格・評価面談にそのまま転用できる形で積み上げます。

## 起動方法

静的ファイルのみ。ビルド不要。

- **公開URL**: <https://bridge-med.github.io/bridge-lp/worklog/>
- **ローカル**: このリポジトリを clone し、`worklog/index.html` をブラウザで直接開く（`file://` でも動作します）。
  - うまく動かない場合はリポジトリ直下で簡易サーバを起動：
    ```bash
    python3 -m http.server 8000
    # → http://localhost:8000/worklog/
    ```
- **スマホ**: 公開URLを開き、ホーム画面に追加（PWA的に1タップ起動）。

データは閲覧している**端末のブラウザ（localStorage）**に保存されます。サーバには送信されません。設定画面から JSON で書き出し／取り込み（バックアップ）できます。

## 画面

| 画面 | 役割 |
|---|---|
| ホーム | 今月件数・実績候補・未計算の数字・レビュー状況・次に追う数字・各追加への導線 |
| 日次ログ | スマホで雑メモを1分入力。必須は日付とメモのみ。「実績ログに変換」で案件ログへ昇格 |
| 案件ログ | 基本情報／下書き(AI補正)／現状整理／数字／打ち手／成果／経営的な意味／再利用できる型／キャリア転用 |
| 数字成果 | Before/After・改善幅・金額換算（未計算は明示）をカード表示。CxOとして語れる成果に整理 |
| 週次レビュー | 今週の日次ログ・実績候補を参照しつつ振り返り。実績候補を案件ログ化する導線つき |
| 月次レビュー | CxOロードマップ上の進捗（取れた経験・足りない経験・来月取りに行く経験） |
| スキルマップ | 現在/目標レベル・ギャップ・取りに行く経験・証明する実績・進捗 |
| 出力 | 職務経歴書用／LinkedIn用／昇格面談用サマリー＋1行実績リスト（スプレッドシート貼付） |

## 使い方の流れ

1. 現場で動いたら **日次ログ** に雑メモ（スマホ）。意味がありそうなら状態を「実績候補」に。
2. 週末に **週次レビュー**。実績候補を **案件ログ** に昇格（ボタンで内容を引き継ぎ）。
3. 案件ログで Before/After・打ち手・経営的な意味・キャリア表現まで詰める。数値は **数字成果** にも登録。
4. 月末に **月次レビュー** でロードマップ進捗を確認。
5. 面談・転職・昇格のタイミングで **出力** からコピペ。

### AI 補正の前提
案件ログには「雑メモ → AI補正後 → 確定ログ」を分けて保存できます。雑メモを ChatGPT / Claude に貼って整形 → 「AI補正後」に貼り付け → 自分で確認して「確定ログ」へ、という運用を想定（アプリ内 AI 連携は今後）。

## 設計（拡張しやすさ）

- `app/schema.js` … 型定義・フォーム定義・マスタ（カテゴリ等）。**データの形の単一の真実**。
- `app/store.js` … 保存層。`collection(all/get/upsert/remove)` に抽象化。**localStorage 実装をここだけ差し替えれば Supabase 等へ移行可能**。初期シードは実データ（リハ単位数改善）のみ。
- `app/app.js` … 画面描画・ルーティング・汎用フォーム生成・出力生成。
- `app/styles.css` … スタイル（白ベース／青グレー／スマホファースト）。

データはフラットなオブジェクトで保持しており、将来テーブル（Supabase）へほぼ1:1で写せます。

## 初期データの扱い（重要）

- 架空の成果は入れていません。初期サンプルは **実在の「リハ部門 平均単位数 19→23 単位」改善のみ**。
- 金額換算が未済のものは「収益改善に寄与した」と断定せず、**「寄与する可能性」「今後金額換算すべき数字」**として保持しています（数字成果では「未計算」と明示）。
- スキルのレベルは初期値を入れていません（あなたが設定する前提）。

不要なら、設定画面の「初期化」でサンプルを消せます。

## プライバシー / セキュリティ（重要）

このアプリは公開リポジトリ（GitHub Pages）でホストされていますが、**入力データは閲覧している端末のブラウザ（localStorage）内だけに保存**され、サーバや GitHub には送信されません。他人が同じURLを開いても見えるのは空のアプリで、あなたのデータは見えません。

ただし運用上の注意：

- **localStorage は平文**（暗号化なし）。共有PC・端末の紛失・他人の操作で見られます。端末ロックと、機種変前の JSON バックアップを。
- **公開リポジトリにコミットされるコード／初期サンプルに固有名詞を書かない**（現状サンプルは一般化済み）。
- **固有名詞は避ける**：病院名・個人名は「A院」等に。具体的な金額・契約条件は入れず、成果は**改善率(%)**で表現するのが安全です（金額換算は任意）。
- **`feed.html` 用の GitHub Issues は公開**。実名・機密は書かないこと。

金額換算について：医療経営支援では保険点数からの金額換算が難しいため、**基本は改善率(%)で記録**します。数字成果は Before/After を数値で入れると改善率を自動計算して表示します。金額は出せるものだけ任意で入れてください。
