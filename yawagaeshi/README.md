# やわ返 🫧

> 送る前に、ひと呼吸。角の立たない返信をつくるアプリ。

LINE・Slack・メール・DM などで、返信に迷ったときに使うスマホ向け Web アプリです。
「相手から来た文」「返したい内容」「相手との関係」「チャネル」「トーン」を入れると、
**やわらかい返信案が3つ**できて、ワンタップでコピーできます。

ChatGPT のような汎用チャットではなく、
**貼る → トーンを選ぶ → 返信案を作る → コピーする** に特化した体験を目指しています。

---

## アプリ概要

- 困っている文を、すぐ貼れる
- 「やわらかく / 丁寧に / 短く / きっぱり…」などのトーンを簡単に選べる
- 返信案が3つ（やわらかめ / ちょうどいい / 短め）出る
- ワンタップでコピー
- 送信前に「冷たく見えないか」「圧が強くないか」を確認できる
- 過去に作った返信を見返せる
- 説教ではなく、相棒キャラ「やわちゃん」がやさしく寄り添う

## 画面

| 画面 | 役割 |
|---|---|
| **ホーム** | クイックアクション（やわらかく返す / 断り文 / 催促文 / 謝罪文）・最近の下書き・相棒の一言 |
| **入力** | 相手から来た文・返したい内容・相手との関係・チャネルを入力（クリップボード貼り付け対応） |
| **トーン選択** | トーン（複数選択）・文章の長さ（単一選択）・現在の設定プレビュー |
| **返信案** | 3案を表示。コピー / もっと丁寧に / もっと短く / お気に入り。各案から送信前チェックへ |
| **送信前チェック** | 冷たさ・圧・長さのメーター＋やさしいコメント・最終メッセージ・コピー・送信（共有シート） |
| **履歴** | 過去の返信を一覧・検索。お気に入りだけの絞り込みも可能 |

下部ナビ：**ホーム / 入力 / 作成 / 履歴**（「作成」は直近の結果、なければ入力へ）

## 起動方法

ビルド不要の静的サイトです。任意の静的サーバーで配信できます。

```bash
cd yawagaeshi
python3 -m http.server 8000
# → http://localhost:8000 をスマホ幅（〜440px）で開く
```

GitHub Pages では `https://bridge-med.github.io/bridge-lp/yawagaeshi/` で公開されます。

> Service Worker は `http://localhost` か `https` 配信時のみ有効です（`file://` 直開きでは無効）。

## ディレクトリ構成

```
yawagaeshi/
  index.html              アプリのシェル（画面コンテナ＋下部ナビ）
  manifest.webmanifest    PWA マニフェスト
  sw.js                   Service Worker（簡易オフラインキャッシュ）
  app/
    styles.css            デザイン（ミント＋淡い青緑＋薄オレンジ・角丸・ダークモード）
    store.js              データモデル / localStorage 永続化 / 履歴・お気に入り / pub-sub
    replyService.js       返信生成のサービス層（モック生成・プロバイダ切り替え式）
    app.js                画面描画 / ルーティング / 相棒キャラ「やわちゃん」
    icon.svg              アプリアイコン
```

UI(`app.js`) は、データを `window.Store`、返信生成を `window.ReplyService` 経由でのみ扱います（責務分離）。

## 返信生成の仕組み（現在はモック生成）

返信文は **`replyService.js`（サービス層）** が生成します。画面側は生成ロジックを一切持ちません。

```js
ReplyService.generateReplies(request)        // → Promise<{ suggestions, riskCheck }>
ReplyService.regenerate(request, label, mode) // もっと丁寧に / もっと短く
```

**現在は外部 AI API を使わない「モック生成」** です。`generateMockReplies()` が、

- 相手との関係（取引先・上司・同僚…）
- チャネル（メール・Slack・LINE…）
- トーン（やわらかく・断る・催促・謝る…）
- 文章の長さ（短め・普通・丁寧に長め）
- 返したい内容（「〜と伝えたい」などのメタ表現を除去し、ていねい調へ軽く整形）

を組み合わせて、**やわらかめ / ちょうどいい / 短め** の3案を出し分けます。
あわせて送信前チェック（`coldness` / `pressure` / `length` を各 1〜3 で見積もり、やさしいコメント）も返します。

> モックなので文面は完璧ではありませんが、「貼る→選ぶ→作る→コピー」という一連の体験が成立することを優先しています。

## 将来の AI API 連携方針（フェーズ2）

API キーを**フロントエンドへ直接埋め込まない**構成を前提にしています。

```
スマホアプリ → 自前API / Edge Function / Serverless → OpenAI / Claude / Gemini
```

`replyService.js` にプロバイダ切り替えの土台を用意済みです。

```js
var REPLY_PROVIDER = 'mock';      // 初期はモック
// var REPLY_PROVIDER = 'openai';  // 将来
// var REPLY_PROVIDER = 'claude';
// var REPLY_PROVIDER = 'gemini';
```

`mock` 以外のときは `callRemote()` が自前 API（`REPLY_ENDPOINT`）を叩く実装に差し替えられます
（現状は未実装のため、通信不可時はモックへフォールバック）。
AI へ投げる想定のシステムプロンプト（角を立てない／意図を変えない／JSON 出力 など）と
ユーザープロンプトの組み立ても `replyService.js` 内に同梱しています。

利用回数制限（例：無料3回/日）は `DAILY_LIMIT` の土台のみ用意しています（現在は無制限）。

## データ保存方式

- 端末内 `localStorage`（キー：`yawagaeshi:v1`）。**アカウント・バックエンド・通信なし**。
- 初回起動時はサンプル履歴を3件投入（催促メール／断り文／資料の確認）。
- 履歴モデル（`ReplyHistoryItem`）:

```ts
type Relationship = "取引先"|"上司"|"同僚"|"部下"|"友人"|"家族"|"その他";
type ReplyChannel = "メール"|"Slack"|"LINE"|"DM"|"その他";
type ReplyTone    = "やわらかく"|"丁寧に"|"短く"|"きっぱり"|"催促"|"断る"|"謝る"|"感情を抜く";
type ReplyLength  = "短め"|"普通"|"丁寧に長め";

type ReplySuggestion = { id; label:"やわらかめ"|"ちょうどいい"|"短め"; text; note?; isFavorite? };
type ReplyRiskCheck  = { coldness; pressure; length; comment };  // 各 1〜3
type ReplyHistoryItem = {
  id; createdAt; originalMessage?; userIntent; relationship?;
  channel; tones[]; length; suggestions[]; selectedText?; riskCheck?;
};
```

## プライバシー上の注意

入力欄や送信前チェック画面に、次の注意を表示しています。

- 個人情報や機密情報は入力しないでください
- 生成された文章は、送信前に必ずご自身で確認してください
- 医療・法律・人事・契約など重要な判断を含む内容は、専門家や責任者にご確認ください

履歴は端末内（`localStorage`）にのみ保存されます。ブラウザのデータ削除や別端末では引き継がれません。
