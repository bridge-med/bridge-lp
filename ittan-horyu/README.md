# いったん保留 🫧

> 買う前に、少しだけ寝かせる。

衝動買いを、やさしくクールダウンするスマホ向けWebアプリ。
欲しいものをすぐ買わずに **24時間 / 3日 / 7日** 寝かせ、期間終了後に「まだ欲しいか」「本当に必要か」を冷静に振り返って、買うか見送るかを判断します。

家計簿（買ったあとに記録する）ではなく、**買う前に一度立ち止まる**ためのアプリです。

---

## アプリ概要

- 今、保留中のものが何件あるか、今月いくら見送れたかをひと目で確認
- 欲しいものを数タップで登録（クールダウン期間つき）
- クールダウン終了後にやさしい問いかけで見直し
- 「見送った金額」が積み上がって、少し達成感がある
- 説教臭くせず、相棒キャラ「ホリュ」がやさしく寄り添う

## 画面

| 画面 | 役割 |
|---|---|
| **ホーム** | 保留中の件数・今月見送れた金額・注目アイテム・相棒の一言 |
| **保留中** | 保留中アイテムの一覧（すべて / 24時間 / 3日 / 7日 / 判断待ち でフィルター） |
| **追加** | アイテム名・価格・カテゴリー・理由（必要/欲しい/勢い）・クールダウン期間を登録 |
| **ふりかえり・判断** | アイテム詳細＋5つの問いかけ → 「買う」/「見送る」。見送ると紙吹雪でお祝い |
| **今月のふりかえり** | 見送り金額・先月比・件数・カテゴリー別の簡易棒グラフ・月切り替え |

下部ナビ：**ホーム / 保留中 / 追加（中央の＋）/ ふりかえり**

## 実装済み機能（必須要件）

1. アイテム追加
2. 保留中アイテム一覧（フィルター付き）
3. クールダウン期間の計算（`reviewAt = createdAt + 期間`）
4. 判断可能状態（`ready`）の自動表示
5. 「買う」/「見送る」のステータス更新
6. 見送った金額の集計（累計・今月）
7. 今月の集計表示（件数・先月比・カテゴリー別グラフ）
8. 下部ナビゲーション
9. スマホ最適化UI（モバイルファースト）
10. ローカル永続化（`localStorage`）

### 追加で実装した機能

- カテゴリー別の簡易アイコン（絵文字プレースホルダー）
- 月別切り替え（前月へさかのぼって閲覧）
- 見送り時の称賛アニメーション（紙吹雪）
- ダークモード対応（OS設定に追従）
- PWA対応（`manifest.webmanifest` + `sw.js`、ホーム追加・簡易オフライン）
- 相棒キャラ「ホリュ」（軽量SVG／状況に応じて表情と一言が変化）
- 写真・スクショ添付（撮影/選択した画像を端末内で縮小して保存し、サムネイル表示）
- 商品ページURLの保存（詳細画面から「商品ページを開く」。http/https のみ許可・新規タブで開く）
- データ管理（「ふりかえり」画面下部）：全部消して空にする／お試しサンプルを入れる

## 起動方法

ビルド不要の静的サイトです。任意の静的サーバーで配信できます。

```bash
cd ittan-horyu
python3 -m http.server 8000
# → http://localhost:8000 をスマホ幅（〜440px）で開く
```

GitHub Pages では `https://bridge-med.github.io/bridge-lp/ittan-horyu/` で公開されます。

> Service Worker は `http://localhost` か `https` 配信時のみ有効です（`file://` 直開きでは無効）。

## ディレクトリ構成

```
ittan-horyu/
  index.html              アプリのシェル（画面コンテナ＋下部ナビ）
  manifest.webmanifest    PWA マニフェスト
  sw.js                   Service Worker（簡易オフラインキャッシュ）
  app/
    styles.css            デザイン（オレンジ＋アイボリー＋淡グリーン・角丸・ダークモード）
    store.js              データモデル / localStorage 永続化 / 集計 / pub-sub
    app.js                画面描画 / ルーティング / 相棒キャラ「ホリュ」
    icon.svg              アプリアイコン
```

`store.js`（データ層）と `app.js`（UI層）を分離。UIは必ず `window.Store` 経由でデータを読み書きします。

## データ保存方式

- 端末内 `localStorage`（キー: `ittan-horyu:v1`）。**アカウント・バックエンド・通信なし**。
- 初回は空からスタート。お試し用サンプルは「ふりかえり」画面下部のボタンから任意で投入できます。
- 添付写真は端末内で長辺900px・JPEG(q0.72)に縮小して data URL で保存（localStorage の容量節約のため）。
- データモデル（`HoldItem`）:

```ts
type ItemStatus    = "holding" | "ready" | "purchased" | "skipped";
type DesireReason  = "必要" | "欲しい" | "勢い";
type CoolingPeriod = "24h" | "3d" | "7d";
type ItemCategory  = "ガジェット" | "服" | "サブスク" | "本" | "家電" | "その他";
type HoldItem = {
  id: string;
  name: string;
  price: number;
  category: ItemCategory;
  reason: DesireReason;
  coolingPeriod: CoolingPeriod;
  createdAt: string;   // ISO
  reviewAt: string;    // ISO（createdAt + クールダウン期間）
  status: ItemStatus;
  memo?: string;
  decidedAt?: string;  // 買う/見送るを判断した時刻（今月集計の基準）
};
```

- `ready` は保存値ではなく**派生状態**：`holding` かつ `reviewAt` を過ぎたものを表示上「判断待ち」として扱います（保存値は `holding` のまま）。

## 注意点

- 通知は OS のローカル通知ではなく **UI上の「判断待ち」表示** で代替しています（Webの制約 + 依頼の代替案に準拠）。
- データは端末ローカルのみ。ブラウザのデータ削除・別端末では引き継がれません。
- 商品画像はアップロード非対応。カテゴリー別の絵文字アイコンで代替しています。
- フォントは Google Fonts を利用（オフライン時はシステムフォントにフォールバック）。
